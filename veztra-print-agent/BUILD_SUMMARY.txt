INSTALLATION & DEPLOYMENT
================================================================================

Windows (5 minutes):
  1. Prerequisites: Node.js 18+, Administrator access
  2. Run: powershell -ExecutionPolicy Bypass -File platforms\windows\install.ps1
  3. Service auto-starts on Windows boot
  4. Config: %APPDATA%\VeztraPrintAgent\config.json
  5. Logs: %APPDATA%\VeztraPrintAgent\logs\service.log

Linux (5 minutes):
  1. Prerequisites: Node.js 18+, sudo access
  2. Run: sudo bash platforms/linux/install.sh
  3. Service managed by systemd
  4. Config: /etc/veztra-print-agent/config.json
  5. Data: /var/lib/veztra-print-agent/veztra.db
  6. Logs: /var/log/veztra-print-agent/service.log

Systemd Commands:
  - Start service:     sudo systemctl start veztra-print-agent
  - Stop service:      sudo systemctl stop veztra-print-agent
  - Check status:      sudo systemctl status veztra-print-agent
  - View logs:         sudo journalctl -u veztra-print-agent -f
  - Enable autostart:  sudo systemctl enable veztra-print-agent

TESTING & VERIFICATION
================================================================================

Health Check:
  curl http://localhost:5050/api/v1/health

Add Printer:
  curl -X POST http://localhost:5050/api/v1/printers \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Test Printer",
      "type": "receipt",
      "brand": "generic",
      "connectionType": "network",
      "host": "192.168.1.100",
      "port": 9100
    }'

Submit Print Job:
  curl -X POST http://localhost:5050/api/v1/print \
    -H "Content-Type: application/json" \
    -d '{
      "printerId": "{printer-id}",
      "data": "Test print\n[CUT]"
    }'

Check Queue:
  curl http://localhost:5050/api/v1/queue/stats

View Logs:
  curl http://localhost:5050/api/v1/logs?lines=50
